## Common Serverpod Concepts

### Core Components
- **Server**: Backend server running Dart
- **Client**: Generated client code for Flutter/Dart apps
- **Protocol**: Serializable classes shared between client and server
- **Endpoints**: API endpoints for client-server communication
- **Database**: Built-in ORM with PostgreSQL support

### Key Patterns
- Protocol files (`.yaml`) define serializable classes
- Endpoint classes handle API logic
- Generated code for type-safe client-server communication
- Database migrations for schema management
- Session management for authentication
